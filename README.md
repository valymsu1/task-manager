# Task Manager

Система управления задачами для команды.

## Установка и запуск

### Backend

```bash
cd backend
npm install
cp .env.example .env
# Отредактируйте .env файл с вашими настройками

# Инициализация базы данных и создание админа
node database.js
node init-admin.js

# Запуск сервера
npm start
# или для разработки с автоперезагрузкой
npm run dev
```

Админ по умолчанию:
- Логин: `admin`
- Пароль: `admin123`

**ВАЖНО:** Смените пароль после первого входа!

### Frontend

```bash
cd frontend
npm install
npm start
```

Приложение откроется на http://localhost:3000

## Роли пользователей

- **Администратор** - полный доступ ко всем функциям
- **Руководитель проекта** - создание задач, назначение исполнителей, управление клиентами
- **Исполнитель** - просмотр и изменение статуса своих задач
- **Наблюдатель** - только просмотр задач и клиентов

## Функционал

- Авторизация по логину и паролю
- Управление пользователями (для админа)
- Управление клиентами
- Создание и отслеживание задач
- Указание дедлайнов и трудоемкости
- Назначение исполнителей и руководителей
- Изменение статусов задач
- Уведомления о новых задачах

## API Endpoints

### Авторизация
- `POST /api/auth/login` - Вход в систему

### Пользователи
- `GET /api/users` - Список пользователей (admin)
- `POST /api/users` - Создать пользователя (admin)
- `PUT /api/users/:id` - Обновить пользователя (admin)
- `DELETE /api/users/:id` - Удалить пользователя (admin)

### Клиенты
- `GET /api/clients` - Список клиентов
- `POST /api/clients` - Создать клиента (admin, manager)
- `PUT /api/clients/:id` - Обновить клиента (admin, manager)
- `DELETE /api/clients/:id` - Удалить клиента (admin)

### Задачи
- `GET /api/tasks` - Список задач (фильтруется по роли)
- `GET /api/tasks/:id` - Получить задачу
- `POST /api/tasks` - Создать задачу (admin, manager)
- `PUT /api/tasks/:id` - Обновить задачу
- `DELETE /api/tasks/:id` - Удалить задачу (admin, manager)

### Уведомления
- `GET /api/notifications` - Список уведомлений
- `PUT /api/notifications/:id/read` - Отметить как прочитанное
- `PUT /api/notifications/read-all` - Отметить все как прочитанные

## Технологии

- **Backend**: Node.js, Express, SQLite, JWT
- **Frontend**: React, React Router
- **Стиль**: Минималистичный белый дизайн

## Перенос на сервер

Для переноса на сервер:

1. Замените SQLite на PostgreSQL/MySQL в production
2. Настройте переменные окружения
3. Используйте nginx для проксирования
4. Настройте SSL сертификаты
5. Настройте email для уведомлений
